# 500+ AI Business Prompts — The Complete Pack

A curated library of prompts organized by use case. Each prompt is designed to get specific, actionable output — not generic AI filler.

---

## BUSINESS STRATEGY

**Strategic decision-making:**
```
I'm trying to decide between [option A] and [option B]. Here are the relevant facts: [facts]. What are the 3 strongest arguments for each side? Then tell me what you'd need to know to make a confident recommendation.
```

**Pre-mortem analysis:**
```
I'm about to [decision]. What are the most common reasons people regret this kind of decision 6 months later? What would I need to verify or believe to feel confident this won't be one of those cases?
```

**Competitive positioning:**
```
Based on what you know about [company/product], what do their most loyal customers love about them that alternatives don't offer? What's the one thing they'd have to screw up to lose those customers?
```

**Hidden assumption finder:**
```
I've been trying to solve [problem] and here's what I've tried: [list]. What assumption am I probably making that's wrong? What would I need to believe for my current approach to be correct?
```

**Steelman both sides:**
```
I believe [position]. Steelman the opposite position as convincingly as possible. Then tell me the weakest part of my own argument.
```

## WRITING & EDITING

**Draft diagnosis:**
```
Read this draft and tell me: where does the reader's attention most likely drop off, and why? Don't rewrite it yet — just diagnose it.
```

**Tone adjustment:**
```
Rewrite this in the style of someone who's slightly annoyed they have to explain it, but still helpful: [your text]
```

**Email: 3 versions:**
```
I need to send an email to [person/role] about [situation]. The outcome I want is [outcome]. The tone I want is direct but not aggressive. Write 3 versions: one too soft, one too aggressive, one that's right. Then tell me which one you'd send and why.
```

**Hard email writing:**
```
I need to [ask for something / say no / push back on something] via email to [person]. Context: [context]. Write a version that's direct without being rude. Then explain your phrasing choices.
```

**Scope change response:**
```
A client just asked for this change to a project we agreed was complete: [describe the change request]. Help me write a professional response that acknowledges the request without committing to it, explains it's outside the original scope, proposes a path forward (quote for the addition, or separate phase), and stays warm. Keep it under 150 words.
```

**Blog intro rewrite:**
```
Here's the intro to my blog post: [paste intro]. Rewrite it so the first sentence makes someone want to read the second one. Don't start with a question. Don't use the word "today."
```

**Make it shorter:**
```
This is too long. Cut it by 40% without losing the key points. Don't add filler to compensate. [paste text]
```

## CODING & DEVELOPMENT

**Error diagnosis:**
```
Here's the error: [error]. Before you suggest a fix, explain in plain English what's actually causing it and why.
```

**Code review:**
```
Do a code review on this code I wrote for a client project: [paste code]. Look for: bugs and edge cases, security issues, performance bottlenecks, things that will be hard to maintain or extend, anything a senior dev would flag in a real review. Be specific about file/function/line where possible.
```

**Learning new concepts:**
```
Explain [concept] to me. Start at the level of someone who understands [adjacent thing I know]. Stop me if I should know something before we continue.
```

**Architecture recommendation:**
```
A client needs to build: [describe the system]. Their constraints: Budget: [estimate], Team: [size/skill level], Timeline: [when they need it], Scale: [expected usage]. Recommend an architecture. Include what to build vs. what to buy, stack recommendation with reasoning, main risks with this approach, and what you'd do differently if budget was 2x. Be opinionated.
```

**Refactoring guide:**
```
Here's a function that works but is hard to read: [paste function]. Explain what it's doing, then refactor it to be cleaner. Don't change the behavior. List what you changed and why.
```

**Test cases I didn't think of:**
```
Here's a function: [paste function]. What edge cases and failure modes am I probably not testing for? Give me the test cases, not just the concepts.
```

**Documentation writer:**
```
Write technical documentation for this codebase/module/feature: [paste code or describe the architecture]. Audience: a developer who knows [language/framework] but hasn't seen this project before. Include: overview (what this does and why it exists), setup, key functions/components, known limitations, where things are likely to break. Format as a README section.
```

**Explain code to non-technical client:**
```
Explain this code to a non-technical client who is paying for it and wants to understand what they got: [paste code or describe the feature]. Explain what it does in plain English, why it matters for their business, and any limitations. Use an analogy if it helps. Keep it under 200 words.
```

## FREELANCING & CLIENT WORK

**Project brief analysis:**
```
I'm writing a proposal for a client project. Here's the brief they gave me: [paste client brief]. Write the "Technical Approach" section of my proposal. It should show I understand the problem, explain what I'll build and why, be specific enough to sound credible but not box me in, be readable by a non-technical founder, and be 200-300 words. Avoid buzzwords.
```

**Scope estimation sanity check:**
```
I'm estimating a freelance project. Here's what the client wants: [describe the project]. My initial estimate is [X hours / $ amount]. Point out scope areas I might be underestimating, hidden complexity that could expand the project, and questions I should ask the client before locking in a price.
```

**Client call prep:**
```
I have a first call with a client building: [describe their project from the brief]. Give me 5 questions I should ask to understand the real scope, 3 red flags to watch for in their answers, and what I should NOT agree to in a first call. I want to come across as experienced and thoughtful.
```

**Difficult feedback delivery:**
```
I need to tell a client that [bad news: delay / problem / limitation]. Help me write a message that's honest about the situation, takes appropriate responsibility without over-apologizing, proposes a clear next step, and keeps the relationship intact.
```

## MARKETING & SALES

**ICP discovery:**
```
Help me identify my ideal customer profile for [product/service]. Ask me 5 questions about my best existing customers (or hypothetical best customers). Based on my answers, draft an ICP description with demographics, psychographics, pain points, and what they're already buying to solve this problem.
```

**Ad copy variants:**
```
Write 5 versions of this headline from most aggressive to most subtle: [describe your offer and audience]. Each version should use a different psychological trigger: scarcity, social proof, fear of loss, curiosity, direct benefit.
```

**Email subject line A/B test:**
```
Write 10 subject lines for this email: [describe the email]. Each should use a different psychological trigger. Include: open rate prediction for each and which one you'd send to the warmest segment vs the coldest.
```

**Landing page critique:**
```
Here's my landing page copy: [paste copy]. Tell me: what's unclear in the first 5 seconds, what objections are unaddressed, what the CTA should be instead, and what single sentence would most improve conversions.
```

**Content repurposing:**
```
Here's a [blog post / email / thread]: [paste content]. Repurpose this into: a 60-second LinkedIn post, 3 tweet variations, a 5-slide presentation outline, and an email newsletter intro. Keep each version appropriate for its platform.
```

## PROJECT MANAGEMENT

**Project brief writer:**
```
I need to write a project brief for [project]. Here's what I know about the goal and constraints: [describe]. Write a 1-page brief that includes: objective (what success looks like), scope (what's in and out), timeline, key stakeholders, risks, and open questions. Keep it scannable — use headers and bullets.
```

**Status update for executives:**
```
Write a project status update for [project name]. Audience: senior executives who want the bottom line in under 2 minutes. Current status: [on track/at risk/blocked]. Key progress: [3 bullet points]. Issues requiring decision: [any blockers]. Next milestone: [date and what it is]. Format: bullet points, no jargon.
```

**Scope document from vague requirements:**
```
A client described their project as: "[vague description]". Turn this into a clear scope of work with: specific deliverables, what's explicitly excluded, assumptions we're making, and 3 questions we need answered before starting.
```

**Retrospective summary:**
```
Here are the notes from our team retrospective: [paste notes]. Summarize into: 3 things we should keep doing, 3 things to stop or change, 2 specific action items with owners and dates. Format for a team Slack post.
```

**Timeline with risk buffers:**
```
I need to create a project timeline for [project]. Key milestones are: [list]. The deadline is [date]. Build a timeline that includes realistic buffers for the riskiest dependencies. Flag where I'm most likely to slip and why.
```

## DESIGN & UX

**Brief clarification:**
```
A client gave me this design brief: "[paste brief]". This is vague. Ask me the 5 most important clarifying questions before I start designing. Then, based on what a typical answer would be, draft a clearer brief I can get approved.
```

**UX copy critique:**
```
Here's the copy for [button / error message / empty state / onboarding flow]: [paste copy]. Tell me: what's unclear, what might cause anxiety or confusion, and write 3 alternative versions ranked by expected conversion.
```

**Persona development:**
```
I'm designing for [describe the target user in 1 sentence]. Based on this, give me a realistic user persona that includes: a day in their life, their top 3 frustrations with current solutions, what they're secretly hoping this product does for them, and what would make them not trust it.
```

**Accessibility audit questions:**
```
I'm designing [describe the component/flow]. What accessibility considerations am I probably forgetting? Give me a checklist of the top 10 things to check, ordered by how often designers miss them.
```

## RESEARCH & ANALYSIS

**Literature synthesis:**
```
I've read several sources on [topic] and here are the key points: [bullet points from your research]. Synthesize these into a coherent summary that identifies: where the sources agree, where they disagree, and what remains genuinely uncertain. Then give me your take on what's most likely true.
```

**Competitor analysis:**
```
I'm building [describe your product]. My main competitors are [list them]. For each competitor, tell me: their core value proposition, who their best customer is, what they do better than alternatives, and where they're weak. Then tell me what gap none of them are filling.
```

**Data interpretation:**
```
Here's some data I'm looking at: [paste data or describe the metrics]. What's the most interesting pattern here? What would you want to investigate further? What explanation are most people probably jumping to that's probably wrong?
```

**Research question sharpener:**
```
I'm trying to answer this question: [your question]. But I think it might be the wrong question. What's the underlying question I should actually be asking? Give me 3 alternatives and explain why each is better or worse.
```

## PERSONAL PRODUCTIVITY

**Overwhelming project breakdown:**
```
I'm overwhelmed by [project]. Break it into the smallest possible steps where each step takes less than 30 minutes. Assume I'm starting from scratch. Give me the first 10 steps in order.
```

**Brilliant friend advisor:**
```
You're my brilliant, blunt friend who has deep expertise in [topic]. I need real advice, not generic tips. Here's my situation: [situation]. What would you actually do? Don't hedge. Don't give me a list of options. Tell me what you'd do.
```

**Goal setting:**
```
I want to achieve [goal] by [date]. I have [current status]. What are the 3-5 leading indicators I should track weekly? What's the most common reason people with my starting point fail to reach this goal? What's the one thing I should do in the next 48 hours?
```

**Week planning:**
```
Here are my priorities for this week: [list]. Here are my constraints: [time available, meetings, etc.]. Build me a realistic weekly plan that protects deep work time, sequences tasks by energy level (hard work in the morning), and includes buffer for the unexpected.
```

**Learning accelerator:**
```
I want to get good at [skill] in [timeframe]. I'm currently at [current level]. What's the 20% of knowledge that gives 80% of the results for this skill? Give me a learning plan with specific resources (not just "read books about it") and a way to test my progress.
```

## OPERATIONS & SYSTEMS

**SOP writer:**
```
Write a standard operating procedure for [task]. Audience: a new team member with no prior knowledge. Format: numbered steps, no assumed knowledge, include what to do if something goes wrong at each step. Keep it under 500 words.
```

**Meeting agenda:**
```
I need to run a [length] meeting about [topic] with [attendees and roles]. The goal is [desired outcome]. Write an agenda with: time allocations, specific questions for each section, and the one decision we must make before leaving the room.
```

**Vendor evaluation:**
```
I'm evaluating vendors for [service/product]. Here are my requirements: [list]. Here are the vendors I'm considering: [list]. Give me: the 5 most important evaluation criteria for this category, red flags to watch for, and the questions I should ask each vendor that will reveal the most.
```
